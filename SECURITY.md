# Security Policy

## 支持范围

安全修复面向默认分支的最新版本。这个项目主要面向单用户、单实例部署，不应被视为多租户身份系统。

## 报告漏洞

优先使用 GitHub 仓库的 “Report a vulnerability” 私密报告入口。若该入口不可用，请创建一个不含利用细节、真实数据或密钥的普通 issue，请维护者建立私密沟通渠道。

请提供受影响版本、复现条件、潜在影响和建议修复。不要在公开 issue 中发布有效凭据、真实对话、数据库内容或可直接利用的细节。

## 部署者责任

- 始终设置强 `APP_PASSWORD` 与稳定的 `FLASK_SECRET_KEY`。
- 使用 HTTPS，并限制应用和数据库的网络访问。
- 把 SQLite 数据库、记忆、上传目录和备份当作敏感数据。
- 只连接可信 MCP 服务，并定期轮换外部服务凭据。
- 移动推送 webhook 必须使用随机密钥校验签名和时间戳；不要把密钥写进仓库、URL 或客户端包。
- 手机 MCP 桥接器应默认只读，在设备端执行权限白名单和逐范围授权；高风险写操作不要仅凭模型调用执行。
- 在更新依赖或公开部署前运行测试和安全扫描。
