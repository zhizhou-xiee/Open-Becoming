# Privacy

Open-Becoming 不内置分析、广告或遥测，但它是一个会把内容发送给外部 AI 与工具服务的应用。部署者负责选择服务商、配置保留策略并向最终用户提供适用的隐私说明。

## 本地保存的数据

默认情况下，以下内容保存在部署实例的本地文件系统：

- SQLite 数据库中的消息、摘要、设置、用量、动态、批注和 MCP 连接信息
- `becoming_memory/` 中的长期记忆与向量索引
- `static/uploads/chat_images/` 中的聊天图片
- SQLite 中的一起听房间、歌词缓存和网易云曲目元数据，以及 `music_library/` 中由部署者上传的本地音乐与封面
- 运行平台产生的访问日志和错误日志

数据库、记忆、上传目录、`.env` 和常见密钥文件不会被 Git 跟踪。不要把运行时数据复制进源码仓库。

## 可能接收数据的外部服务

根据配置和使用方式，数据可能发送到：

- OpenRouter：角色对话、摘要及可选的记忆打标
- Anthropic：配置为 Anthropic 的角色对话
- DeepSeek：可选的记忆元数据分析
- Google Gemini API：可选的记忆语义向量
- 用户添加的 MCP 服务：执行被启用的自定义工具
- 用户配置的移动推送 webhook：接收主动消息的角色标识、消息 ID 和不超过 240 字符的预览
- 网易云音乐：搜索、歌单、歌词、封面和播放请求；配置账号时还会携带部署者的 `MUSIC_U` Cookie
- Google Fonts：浏览器加载字体和 Material Symbols 时会发起网络请求

发送内容可能包括当前消息、历史消息、摘要、召回的记忆、图片、工具参数和必要的角色人设。具体保留和训练政策由相应服务商决定。

## 凭据

- 模型 API key 从服务端环境变量读取，不应返回给浏览器。
- 模型 API key 不写入 SQLite、HTML 或前端 JavaScript；角色配置接口只返回模型名称和提供商，不返回凭据。
- 自定义 MCP token 存在 SQLite 数据库中，接口响应会省略 token，但数据库备份仍应视为敏感文件。
- 移动推送 URL 和签名密钥来自环境变量，不会由 `/api/mobile/extensions` 返回。接收端仍可能保存通知内容，应由部署者自行配置保留策略。
- 网易云 `MUSIC_U` 只从服务端环境变量读取，不写入 SQLite 或前端响应；它等同登录凭据，日志、截图和备份脚本都不应记录。
- 网页版一起听已内置；语音、原生手机播放器和手机查询仍需相应伴侣端或 MCP 服务。语音扩展约定默认不由核心应用保存音频。
- `APP_PASSWORD` 只适合单用户访问控制，不代替 HTTPS、平台身份验证或网络隔离。

## 发布前检查

1. 使用全新的数据库和记忆目录。
2. 设置强 `APP_PASSWORD` 和稳定、随机的 `FLASK_SECRET_KEY`。
3. 只启用可信的模型和 MCP 服务。
4. 手机查询服务必须在设备端限制数据范围、结果条数和角色权限，不能只依赖模型提示词。
5. 移动推送接收端必须校验 HMAC 签名和时间戳，并避免长期保存消息预览。
6. 检查平台日志是否会记录请求正文或凭据。
7. 配置 HTTPS、备份、访问控制和数据删除流程。
8. 若需完全离线，移除 Google Fonts 链接并自托管所需字体与图标。

## 删除数据

停止应用后，删除部署者指定的 SQLite 数据库、记忆目录、图片上传目录和音乐目录即可移除本实例的主要持久化数据。外部服务已经接收的数据需按相应服务商的流程处理。
