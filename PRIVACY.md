# Privacy

Open-Becoming 不内置分析、广告或遥测，但它是一个会把内容发送给外部 AI 与工具服务的应用。部署者负责选择服务商、配置保留策略并向最终用户提供适用的隐私说明。

## 本地保存的数据

默认情况下，以下内容保存在部署实例的本地文件系统：

- SQLite 数据库中的消息、摘要、设置、用量、动态、批注和 MCP 连接信息
- `becoming_memory/` 中的长期记忆与向量索引
- `static/uploads/chat_images/` 中的聊天图片
- 运行平台产生的访问日志和错误日志

数据库、记忆、上传目录、`.env` 和常见密钥文件不会被 Git 跟踪。不要把运行时数据复制进源码仓库。

## 可能接收数据的外部服务

根据配置和使用方式，数据可能发送到：

- OpenRouter：角色对话、摘要及可选的记忆打标
- Anthropic：配置为 Anthropic 的角色对话
- DeepSeek：可选的记忆元数据分析
- Google Gemini API：可选的记忆语义向量
- 用户添加的 MCP 服务：执行被启用的自定义工具
- Google Fonts：浏览器加载字体和 Material Symbols 时会发起网络请求

发送内容可能包括当前消息、历史消息、摘要、召回的记忆、图片、工具参数和必要的角色人设。具体保留和训练政策由相应服务商决定。

## 凭据

- 模型 API key 从服务端环境变量读取，不应返回给浏览器。
- 自定义 MCP token 存在 SQLite 数据库中，接口响应会省略 token，但数据库备份仍应视为敏感文件。
- `APP_PASSWORD` 只适合单用户访问控制，不代替 HTTPS、平台身份验证或网络隔离。

## 发布前检查

1. 使用全新的数据库和记忆目录。
2. 设置强 `APP_PASSWORD` 和稳定、随机的 `FLASK_SECRET_KEY`。
3. 只启用可信的模型和 MCP 服务。
4. 检查平台日志是否会记录请求正文或凭据。
5. 配置 HTTPS、备份、访问控制和数据删除流程。
6. 若需完全离线，移除 Google Fonts 链接并自托管所需字体与图标。

## 删除数据

停止应用后，删除部署者指定的 SQLite 数据库、记忆目录和上传目录即可移除本实例的主要持久化数据。外部服务已经接收的数据需按相应服务商的流程处理。
