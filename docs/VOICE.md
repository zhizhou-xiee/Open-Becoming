# 说说喵°语音收发

Open-Becoming 的语音是一个默认关闭、供应商无关的可插拔功能。它分成两条互不依赖的链路：

- TTS（发语音）：模型在合适时调用 `send_voice`，服务器把文字交给语音服务，再发送带文字稿的 AI 语音消息。
- STT（收语音）：用户在手机上主动录音，服务器先转成文字，再把文字交给原有聊天流程。

它适合已经有 OpenAI-compatible 语音 API，或愿意自己托管 HTTP 语音服务的个人部署者。不配置、不启用时不会申请麦克风权限，也不会向模型暴露 `send_voice`。

## 开始前

1. 先完成 Open-Becoming 的普通部署，并确认文字聊天正常。
2. 生产环境使用 HTTPS。iPhone 和多数现代浏览器只会在 HTTPS 页面（或本机 `localhost`）提供麦克风录音。
3. 准备 TTS/STT 服务的地址、token、模型名和可用音色 ID。
4. 只使用自己有权使用的音色，不要未经许可克隆他人声音。聊天气泡会明确标记“AI 语音”。

登录后打开「更多 → 说说喵°语音收发」。所有开关默认关闭，建议先配置 TTS 并试听，再配置 STT。

## 配置 TTS（模型发语音）

### OpenAI-compatible

选择“OpenAI-compatible”，填写：

- 地址：完整的语音合成接口，例如 `https://api.openai.com/v1/audio/speech`。
- Token：服务商发放的密钥。留空保存会保留已有 token；勾选“清除已保存 Token”才会删除。
- 模型：例如 `gpt-4o-mini-tts`，以你的服务商文档为准。
- 格式：选择服务支持的输出格式。
- 角色音色：为 `char1`–`char6` 分别填写服务商支持的 `voice_id`；空白角色不会获得语音工具。

服务器发送 JSON：

```json
{
  "model": "gpt-4o-mini-tts",
  "input": "这是一条准备合成的角色文字。",
  "voice": "alloy",
  "response_format": "mp3"
}
```

响应应直接返回音频内容，并给出正确的 `Content-Type`。

### 自定义 HTTP TTS

选择“自定义 HTTP”时，服务器会向配置地址发送 Bearer token（如果填写）和以下 JSON：

```json
{
  "text": "这是一条准备合成的角色文字。",
  "model": "your-model",
  "voice_id": "your-voice-id",
  "response_format": "mp3"
}
```

自定义服务可直接返回音频，也可返回 JSON：

```json
{
  "audio_base64": "BASE64_AUDIO_DATA",
  "mime_type": "audio/mpeg"
}
```

配置完成后先点“保存”，再点“试听”。试听会使用当前选中的角色音色，并计入当天次数与估算费用。

## `send_voice` 怎样工作

同时满足以下条件时，角色模型才会得到 `send_voice` 工具：

1. 语音总开关已开启；
2. 当前角色填写了 `voice_id`；
3. 当天次数和估算费用尚未到上限。

模型可以正常回复文字，也可以在适合时调用 `send_voice`。调用中的文字要经过字数与每日限额检查；合成失败时仍保留普通文字回复，不会让整轮聊天消失。成功后，音频保存在 SQLite，聊天中显示紧凑的播放气泡；折叠的文字稿会进入对话历史、摘要和长期记忆链路。

## 配置 STT（用户收发语音）

开启“收语音 / STT”后，可以单独选择服务商：

### OpenAI-compatible

- 地址：完整的转写接口，例如 `https://api.openai.com/v1/audio/transcriptions`。
- 模型：例如 `gpt-4o-mini-transcribe`，以服务商文档为准。
- Token：可以单独填写，也可以选择复用 TTS 已保存的 Token。
- 最大上传：限制单次录音文件大小；服务器还有 20 MB 硬上限。

服务器用 `multipart/form-data` 发送 `file`、`model` 和 `response_format=json`，并读取响应中的 `text`。

### 自定义 HTTP STT

自定义 STT 同样接收 `multipart/form-data`，字段为 `file` 和 `model`；响应可以是：

```json
{"text": "转写后的文字"}
```

也可以直接返回纯文本。录音只用于这一次转写，Open-Becoming 不持久化原始 STT 文件；语音服务商是否留存由其政策和你的配置决定。

保存后，单聊输入框左侧会出现麦克风按钮。点一次开始、再点一次结束；最长约 60 秒。浏览器完成上传和转写后，会把文字填入输入框并走原有发送流程。若当前浏览器不支持直接录音，会退回到选择音频文件。

## 限额与费用保护

设置面板可配置：

- 单条最大字数；
- 每日 TTS 次数（包括试听）；
- 每 1000 字估算费用；
- 每日估算费用上限。

项目不会写死任何供应商价格。费率默认为 `0`，表示不做金额估算，但字数和每日次数仍然生效。请按供应商当前账单单位手动填写估算费率；这只是应用侧保护，不替代服务商账户中的硬预算和告警。

## Token、数据与备份

- Token 从设置页通过你自己的 HTTPS 站点提交一次，保存在服务端 SQLite；读取配置时只返回“已配置”，不会回显原文。
- 不要把 token 写进仓库、截图、浏览器公开环境变量或 Issue。
- SQLite 备份包含语音 token、TTS 音频、文字稿和用量记录，必须像密钥文件一样保护。
- 删除相应聊天消息会同时删除它的 TTS 音频；删除整个数据库会移除本实例保存的语音配置、token 和音频。
- 轮换 token 时直接填入新值并保存；只有明确勾选清除项才会移除旧值。

## 反向代理与常见问题

### 看不到麦克风按钮

确认语音总开关和 STT 开关都已开启并保存。iPhone 还需要用 HTTPS 打开站点，并在 Safari 的网站设置中允许麦克风。

### 能录音但转写失败

检查 STT 地址是否是完整接口、token 和模型是否有效，以及服务是否接受 iPhone 常见的 `audio/mp4`。反向代理也要允许音频上传；单次文件不能超过面板设置和 20 MB 硬上限。

### 试听返回 401/403

重新填写 token 并保存。配置接口不会返回旧 token，因此浏览器开发工具里也无法用读取设置的方式找回它。

### 自定义服务返回成功但没有声音

直接音频响应必须有正确的音频 `Content-Type`；JSON 响应必须包含 `audio_base64`（或 `audio`）以及建议的 `mime_type`。不要返回本地文件路径或需要另一套登录态的临时 URL。

### 角色一直不发语音

确认该角色有 `voice_id`，每日限额尚未用完。`send_voice` 是模型可选择的工具，不保证每条消息都调用；若需要测试供应商连通性，请使用“试听”。

### 自托管地址连不上

地址必须是 `http://` 或 `https://`。公开部署建议使用 HTTPS；内网 HTTP 只适合你能控制网络边界的自托管环境。多副本部署会让进程内的即时限额检查存在短暂竞争，本项目仍建议单实例运行。
